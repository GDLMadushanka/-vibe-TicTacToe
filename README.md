# Tic Tac Toe — VS Code Extension

A polished Tic Tac Toe game playable directly inside VS Code's sidebar. Challenge the AI at two difficulty levels

Medium - beatable, or give it your best shot on Hard mode, where the AI never loses.

## Features

- **Difficulty selection** — choose Easy or Hard before every game
- **Easy mode** — AI plays strategically but makes realistic blunders; beatable with good play
- **Hard mode** — AI uses Minimax with Alpha-Beta Pruning; mathematically unbeatable
- **Persistent scoreboard** — tracks Player wins, AI wins, and draws across games
- **Animated UI** — pop-in moves, win-line draw animation, gold pulse on winning cells
- **VS Code notifications** — toast messages for win / loss / draw results

## Prerequisites

| Tool | Version |
|------|---------|
| [Node.js](https://nodejs.org/) | 18 or later |
| [npm](https://www.npmjs.com/) | 9 or later (bundled with Node.js) |
| [VS Code](https://code.visualstudio.com/) | 1.94 or later |

## Installation

```bash
git clone https://github.com/GDLMadushanka/-vibe-TicTacToe.git
cd -vibe-TicTacToe
npm install
```

## Build

Compile the TypeScript source to JavaScript:

```bash
npm run compile
```

To watch for changes and recompile automatically during development:

```bash
npm run watch
```

## Run

1. Open the project folder in VS Code:
   ```bash
   code .
   ```
2. Press **F5** (or go to **Run → Start Debugging**) to launch the **Extension Development Host** — a second VS Code window with the extension loaded.
3. In the Extension Development Host window, click the **Tic Tac Toe icon** in the Activity Bar (left sidebar).
4. The game panel opens. Select a difficulty and start playing!

## How to Play

1. When the game loads, a **difficulty screen** appears — pick **Easy** or **Hard**.
2. You play as **X** (red) and always go first. The AI plays as **O** (cyan).
3. Click any empty cell to make your move.
4. After each game, click **New Game** to return to the difficulty screen.

### Difficulty levels

| Mode | Behaviour |
|------|-----------|
| **Easy** | Takes wins ~70% of the time, blocks your wins ~60% of the time, plays randomly 40% of the time — beatable with a good strategy |
| **Hard** | Uses the Minimax algorithm with Alpha-Beta Pruning — every move is optimal; the best outcome you can force is a draw |
